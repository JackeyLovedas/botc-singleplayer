## abilityRules

外部来源一致支持以下规则主张：

1. Dreamer 每晚选择除自己和 Traveller 之外的一名玩家。
2. Dreamer 得知一个 GOOD 角色和一个 EVIL 角色。
3. 两个角色中恰好一个是目标玩家的真实角色。
4. GOOD 角色类别是 Townsfolk 或 Outsider；EVIL 角色类别是 Minion 或 Demon。
5. 角色的 GOOD/EVIL 分类来自角色固有 character type/default alignment，而不是目标玩家后来改变的个人 alignment。
6. 本 Slice 的目标真实角色取行动结算时 `CurrentCharacterState`。
7. 打开机会或选择目标本身都不等于交付信息；只有持久化的信息 delivery 是已交付历史事实。

正常、有效、无 Vortox 路径可以安全复用 `DREAMER_FALSE_ROLE_POLICY_VERSION = dreamer-false-role-policy-v1` 与 `dreamer-information-model-v1` 的正常信息策略，但不得复用 V1 opportunity discriminator，也不得把该策略用于 `SOURCE_IMPAIRED` 或 Vortox 路径。

确定性模拟器可在合法反侧角色中按稳定 code-unit 顺序选择最小 role ID，且不得依赖 catalog 输入顺序、locale 或随机数。这是官方 Storyteller 合法裁量范围内的确定性产品子集，不是官方规定的唯一选择。合法反侧候选不足时 fail closed。

## interactions

### Target legality

- source 不得选择自己；
- source 只能选择当前建模 roster 中的其他合法玩家；
- Traveller 不属于本 Slice 的可选目标模型，不能用 Traveller 路径扩大范围。

### Target character truth

- 如果目标在结算前发生角色变化，答案使用结算时的新 `CurrentCharacterState`；
- 如果目标在 delivery 后发生角色变化，历史信息不回算、不改写；
- 仅玩家 alignment 改变不改变目标的真实角色，也不改变按角色固有 type 生成的 GOOD/EVIL答案。

### Source effectiveness

正常信息只在 canonical state 证明下列事实时可交付：source 是当前 base Dreamer、unique active Dreamer tenure 有效、canonical base ability instance 有效、当前角色仍是 Dreamer、没有作用于 source 的 represented DRUNK/POISONED impairment、当前没有有效 Vortox，且不存在其他已知但无法解析的 source-effect constraint。

### Ledger and replay

有效正常 delivery 产生一个 terminal ledger fact：

- `outcomeStatus = NORMAL`
- `causeKind = NO_OTHER_CHARACTER_ABILITY`
- `causedByAnotherCharacterAbility = false`

Settlement 不产生第二个 fact。事实必须绑定 source、task、player、target 与 revision。Later Mathematician 可以读取此 NORMAL fact，但它不计入数学家数字；本 Slice 不改变 2B18B 公共合同。

Target、delivery 和 settlement 必须作为同一原子批次按固定顺序持久化。历史 delivery 必须保留交付时事实，不能从后来的角色状态重新计算。

## drunkennessRules

醉酒角色没有能力，但可能收到任意信息。本 Slice 不实现这种信息生成或 Storyteller 裁量。

当 canonical represented state 表明 source 受 `DRUNK` 影响时，2B19A2 必须 retryable、receipt-free fail closed：不产生 target、delivery、settlement、ledger fact，不关闭 opportunity，不改变 game version。该边界不得被解释为醉酒信息已实现。

## poisoningRules

中毒角色没有能力，但可能收到任意信息。本 Slice 不实现这种信息生成或 Storyteller 裁量。

当 canonical represented state 表明 source 受 `POISONED` 影响时，2B19A2 必须 retryable、receipt-free fail closed，且没有任何领域写入。

No Dashii 的邻座 Townsfolk 中毒推导尚未实现。当当前 Demon 是 No Dashii 且模型不能证明 base Dreamer 不受影响时，必须同样 fail closed；不得假设 source 有效、不得实现 No Dashii poison engine，也不得把 No Dashii 简化为一个普通 POISONED marker。

## VortoxRules

外部规则要求有效 Vortox 下 Townsfolk 能力产生虚假信息。对 Dreamer 来说，强制虚假意味着展示的两个角色都不能是目标真实角色。

2B19A2 不实现这一信息路径。若当前存在有效 Vortox，必须 retryable、receipt-free fail closed，不产生 target、delivery、settlement、ledger 或 opportunity close。Vortox forced-false 保留给 2B19A3，不得将正常路径错误复用到该状态。

## characterChangeRules

### Source

Base source 必须由 accepted history 中当前角色、唯一 active Dreamer tenure 和 canonical base ability instance 共同证明。旧 assignment、旧 tenure、caller-supplied source 或 Philosopher-gained holder 不能替代当前 base source。

### Target

目标在结算前改变角色时，真实角色是 settlement-time `CurrentCharacterState`；delivery 后的角色变化不得修改历史 delivery 或 ledger fact。

本 Slice 不实现角色变化 producer、通用角色变化 task 重建或 gained Dreamer。

## alignmentChangeRules

玩家 alignment 与角色能力身份、角色固有 GOOD/EVIL 分类必须分离。仅 alignment 改变：

- 不改变 source 的 Dreamer 能力身份；
- 不改变 target 的真实角色；
- 不改变基于角色固有 character type/default alignment 的 GOOD/EVIL答案。

本 Slice 不实现 alignment change producer 或 history mutation。

## storytellerDiscretion

官方规则允许 Storyteller 从合法反侧角色中选择要展示的假角色。2B19A2 不实现自由选择、候选集持久化、最终 Storyteller choice record 或任意信息。

稳定最小合法 role ID 只是本项目当前 accepted deterministic simulator policy，且仅用于正常、有效、无 Vortox路径。它不得宣称为官方唯一解释。

## productCompatibilityClaims

以下是产品合同，不是外部 BOTC 规则：

1. Accepted V1 history 保持 exact replay。
2. Accepted V2-plan + legacy-V1 history 保持 exact replay。
3. Slice 2B19A1 `dreamer-first-night-action-opportunity-v2` exact shape 永久不变，保持 replay-only。
4. 新 producer 生成独立版本的 actionable opportunity；不得原地修改 V2、添加 optional 字段或迁移旧 event。
5. 同一 task 的多 generation、duplicate opening、mixed history 必须 fail closed。
6. 正常成功批次固定为 target chosen、information delivered、scheduled task settled，并只增加一次 game version。
7. 成功后 phase 保持 `FIRST_NIGHT`，opportunity 从 OPEN 变为 CLOSED，不产生 `PhaseTransitioned`。
8. Retryable dependency/effectiveness/append/prospective/metadata failure 不写 receipt，并允许同一 command ID 在依赖恢复后重试。
9. 只有 source player 及其 AI private view 可见 target、goodRole、evilRole、knowledgeStage、knowledgeModelVersion。
10. Projection 不得泄漏真实角色标记、真实 alignment、哪一个为真、candidate set、effectiveness、impairment/Vortox 检查、ledger、tenure、ability instance、assignment 或 `CurrentCharacterState`。
