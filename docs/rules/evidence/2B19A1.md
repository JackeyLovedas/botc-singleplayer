## abilityRules

### External rule claims

1. Dreamer 是 Townsfolk。
2. Dreamer 的能力文本为：每个夜晚选择一名不是自己或 Traveller 的玩家，然后得知一个善良角色和一个邪恶角色，其中一个是目标的真实角色。
3. “Each night”包含首夜，因此 base Dreamer 在首夜的官方 Dreamer 位置行动。
4. 对 base Dreamer 路径，采取行动的规则主体是持有当前 base Dreamer 角色及能力的玩家。
5. “不能选择自己或 Traveller”是真实 Dreamer target 规则，但 target choice 不属于本 Slice，不得通过 opportunity foundation 声称已经实现。
6. 官方规则不定义 `V1`、`V2`、opportunity schema、task ID、opportunity ID、role tenure ID 或 ability-instance ID。

### Product compatibility/contract claims

1. Base Dreamer V2 opening opportunity 是产品在正式 target/action 之前建立的待行动合同，不是官方规则中的独立能力结果。
2. Opening opportunity 本身：
   - 不选择 target；
   - 不产生 Dreamer information；
   - 不选择 GOOD/EVIL candidate；
   - 不执行 Storyteller 裁量；
   - 不形成 terminal ability outcome；
   - 不写入 Dreamer ledger fact；
   - 不增加 Dreamer 私有知识。
3. Base source 的 canonical source contract 应绑定当前 base Dreamer 玩家、座位、base role tenure、Dreamer ability instance 和对应 Dreamer task。这是产品 provenance 合同，不是新增 BOTC 规则。
4. V1 accepted history 必须继续精确 replay。V1/V2 payload 和历史隔离是事件兼容性要求，不是外部规则修正。
5. 新 V2 opportunity 尚未具有 target/action resolver 时，提交必须 receipt-free fail closed；这属于产品安全合同。
