## evidenceScope

本文件只为现有 canonical role-tenure derived state 增加 Dreamer 身份持续区间提供规则门禁。

- **外部 BOTC 规则**：玩家当前拥有的角色、角色能力的行动时机，以及角色变化会改变玩家当前角色；
- **产品派生状态合同**：用 canonical tenure 记录某玩家从哪个 revision 开始、到哪个 revision 结束持有某角色。

`role tenure`、tenure ID、revision、replay validator 和 transition fact 都不是官方 BOTC 术语。本 Slice 只允许把它们作为 accepted event stream 的可重建身份索引，不能让 tenure 决定能力是否有效、信息真假或任何能力结果。

本 Slice不实现 Dreamer opportunity、target、information、candidate、delivery、ledger、projection、Vortox、impairment 或 gained Dreamer。

## abilityRules

### External rule claims

1. Dreamer 是 Townsfolk。
2. Dreamer 的能力以“Each night / 每个夜晚”开始；夜间行动主体是当时持有 Dreamer 角色或合法获得 Dreamer 能力的玩家。
3. 官方运行说明要求每夜唤醒 Dreamer，因此执行 base Dreamer 能力时必须能够确定对应的当前角色持有者。
4. Snake Charmer 是会改变角色与阵营的官方能力：命中 Demon 时，相关玩家的当前角色和阵营会发生变化。
5. 官方规则描述当前角色和角色变化，但不定义 repository 的 role-tenure schema、tenure ID 或 revision grammar。

### Product derived-state claims

1. Initial `CharactersAssigned` accepted history可以派生 base Dreamer tenure，表示该玩家从初始角色 revision 开始持有 Dreamer。
2. 若 accepted canonical character transition 的 before/after role 涉及 Dreamer：
   - 非 Dreamer → Dreamer：建立新的 Dreamer tenure；
   - Dreamer → 非 Dreamer：结束旧 Dreamer tenure；
   - Dreamer → 另一 tracked role：结束 Dreamer tenure并按同一权威角色域建立新 tenure；
   - 另一 tracked role → Dreamer：结束旧 tracked tenure并建立 Dreamer tenure。
3. Tenure只记录角色身份持续区间及其 accepted provenance。
4. Tenure不决定：
   - Dreamer 能力是否有效；
   - 是否醉酒或中毒；
   - Vortox 是否有效；
   - Dreamer target是否合法；
   - Dreamer 信息真假；
   - Storyteller选择；
   - ability outcome。
5. 本 Slice只能扩展现有 derived role-tenure state，不能增加新的 domain event、event payload字段、GameState 顶层 tenure字段或第二套 Dreamer 专用 tenure系统。

## interactions

1. Dreamer 的夜间能力需要角色/能力来源身份，但 tenure本身不产生能力。
2. Snake Charmer证明角色与阵营可以在游戏过程中发生 canonical change，且历史角色不能只由最新角色快照替代。
3. Snake Charmer官方交换本身只在 Snake Charmer、Demon及 Philosopher-gained Snake Charmer边界内改变角色；不得声称它单独创造 Dreamer。

5. Pit-Hag、Barber及其他角色变化 producer的能力实现、命令和事件均不在本 Slice扩展。
6. Philosopher-gained Dreamer是能力获得，不是角色变为 Dreamer，不得错误创建 base Dreamer tenure。

## characterChangeRules

### External rule boundary

角色变化会改变玩家的当前角色；历史不能被后来角色覆盖。Snake Charmer官方页面明确展示同一玩家在一次结算前后具有不同角色。

### Product derived-state representation

1. Initial Dreamer assignment从 revision `1` 建立 base tenure。
2. 进入 Dreamer的 accepted canonical transition从其 `nextCharacterStateRevision` 建立新 tenure。
3. 离开 Dreamer的 accepted canonical transition在对应 revision结束旧 tenure。
4. Tenure区间为历史身份索引；查询某 revision时必须使用覆盖该 revision的唯一合法区间。
5. 同一玩家同一 revision不得有两个 active tracked-role tenure。
6. Duplicate、overlap、orphan、invalid provenance和不连续 derived state必须 fail closed。
7. 不得从 caller-supplied transition、测试 fixture或最新角色快照伪造历史。
8. 不得新增角色变化命令、事件或 payload字段。

## alignmentChangeRules

角色与阵营是分离维度。

- 单独阵营变化不开始或结束 Dreamer role tenure；
- Dreamer tenure不得存储或推断阵营；
- Snake Charmer同时交换角色和阵营时，角色 tenure只跟随 canonical character transition；
- 后续 alignment change不得重写早先 role tenure；
- 本 Slice不实现 alignment producer或alignment history mutation。

## storytellerDiscretion

本 Slice没有 Storyteller能力选择。

Storyteller不选择 tenure ID、revision或区间。它们必须由 accepted history确定性派生。

Dreamer真假角色候选、信息选择及其他 Storyteller裁量全部不在本 Slice。

## requiredRegressionTests

### Tracked role domain and ID grammar

1. 权威 tracked role domain精确包含：
   - `cerenovus`
   - `dreamer`
   - `mathematician`
   - `philosopher`
   - `seamstress`
   - `vortox`
2. TypeScript union、runtime predicate、formatter、parser、bootstrap、transition reconciliation和validator使用同一权威角色域。
3. Dreamer tenure formatter/parser exact round-trip。
4. 接受 canonical Dreamer tenure ID。
5. 拒绝未知 role。
6. 拒绝 `Dreamer`大小写别名、尾随空格和extra segment。
7. 拒绝非canonical seat及revision前导零别名。
8. 现有五个 tracked role的ID保持不变。

### CharactersAssigned bootstrap

9. Initial base Dreamer assignment确定性建立一条 tenure。
10. `acquiredCharacterStateRevision = 1`。
11. `startedBy.kind = CHARACTERS_ASSIGNED`。
12. source event ID、sequence、player、seat和role与真实 assignment一致。
13. 非tracked role不建立 tenure。
14. Duplicate assignment或duplicate tenure fail closed。
15. Expected initial tenure set exact validation包含 Dreamer。

### Transition reconciliation

16. Accepted 非Dreamer → Dreamer transition建立新 tenure。
17. Accepted Dreamer → 非tracked role transition结束旧 tenure且不建立非tracked tenure。
18. Accepted tracked role → Dreamer结束旧 tenure并建立 Dreamer tenure。
19. Accepted Dreamer → tracked role结束 Dreamer tenure并建立新 tracked tenure。
20. acquired/ended revision与真实 character-state revision一致。
21. 同一 transition只处理一次。
22. orphan processed transition ID被拒绝。
23. orphan transition-started tenure被拒绝。
24. duplicate transition、duplicate active tenure和overlapping interval被拒绝。
25. transition provenance与 accepted event、player、seat、before/after role及revision一致。

### Active-tenure queries

26. Initial Dreamer在 revision 1 active。
27. 进入 Dreamer前 inactive，从 acquired revision开始 active。
28. 在 ended revision及之后 inactive。
29. continuous-range查询遵循完整区间边界。
30. 无匹配时保持现有 fail-closed/undefined合同。
31. 两条匹配 active Dreamer tenure时 fail closed。

### Compatibility and non-effects

32. Accepted旧历史继续 replay。
33. 新 rebuild从现有 accepted event stream确定性补建 Dreamer derived tenure。
34. 不修改任何 accepted event payload或event type。
35. 不新增第二套 tenure state。
36. Seamstress及其他现有 tracked-role tenure regression保持不变。
37. Dreamer tenure不受单独alignment change影响。
38. 醉酒、中毒和Vortox状态不改变 tenure，也不由 tenure推断。
