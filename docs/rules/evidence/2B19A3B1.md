## abilityRules

1. Dreamer 每夜选择一名玩家，但不能选择自己或旅行者。
2. Dreamer 得知恰好两个角色：
   - 一个原生 GOOD 角色，即镇民或外来者；
   - 一个原生 EVIL 角色，即爪牙或恶魔。
3. 在普通有效信息下，两个角色中恰好一个是目标的真实角色。
4. Philosopher 每局一次选择一个善良角色并获得其能力；Philosopher 不因此成为该角色。
5. 如果所选角色已经在场，原角色玩家醉酒。
6. Philosopher 选择 Dreamer 时：
   - Philosopher 获得 Dreamer 能力；
   - 原生/base Dreamer 成为 DRUNK；
   - Philosopher 在 Dreamer 通常行动的位置使用获得的能力。
7. 当前 Slice 只处理原生/base Dreamer 的表面结算，不处理 Philosopher-gained Dreamer 的结算。
8. 生效的 Vortox 要求 Townsfolk 能力产生的信息为假。
9. Vortox 的约束在信息来源醉酒或中毒时仍然适用。
10. 因此当前成功边界中的表面 Dreamer 结果必须保持一个 GOOD 加一个 EVIL，同时两者都排除目标真实角色。

## drunkennessRules

1. 醉酒玩家没有能力。
2. 说书人仍应让玩家经历看似正常的能力流程，使其不知道自己醉酒。
3. 信息角色醉酒时，可以收到正确或错误信息，但信息仍需保持该角色能力允许的形状。
4. Vortox 存活且有效时，醉酒 Townsfolk 的表面信息仍必须为假。
5. Philosopher-produced DRUNK 是当前路径的正向状态事实。
6. 如果产生该 DRUNK 的 Philosopher 失去能力，持续醉酒效果结束。
7. 当前 Slice 只接受一个 exact canonical Philosopher-produced DRUNK；缺失、不匹配、重复或冲突的 impairment 不属于成功规则边界。

1. 官方规则中，中毒与醉酒在“玩家没有能力，但表面流程可继续”方面机械效果相同。

3. 但当前 bounded Slice 没有已批准的 canonical base-Dreamer POISONED producer 成功路径。
4. 因此：

= R4_FUTURE_HYPOTHETICAL_STATE

6. 当前 Slice 不得以 fixture、caller-provided impairment 或推测状态代替 canonical producer。

## VortoxRules

1. Vortox 的 Townsfolk false-information 能力只在 Vortox 存活且有效时适用。
2. 死亡玩家通常立即失去能力；死亡 Vortox 不继续施加该约束。
3. 醉酒或中毒的 Vortox 没有能力，因此不施加 Vortox false-information 约束。
4. 当前成功路径只处理一个可证明的、当前的、存活且有效的 Vortox。
5. Vortox 不改变 Dreamer 输出形状。
6. Vortox 不把角色变化或阵营变化通知变成虚假信息。
7. 当前 Slice 不处理死亡、醉酒、中毒、重复、陈旧、冲突或身份不可证明的 Vortox 成功路径。

## alignmentChangeRules

1. 角色的原生 GOOD/EVIL 类型与玩家当前阵营相互独立。
2. Dreamer 的一 GOOD、一 EVIL 指角色类型，不是目标玩家当前阵营。
3. 只改变目标阵营不会改变目标真实角色，也不会改变需要从 pair 中排除的角色身份。
4. 玩家获知自身角色或阵营变化不是通常意义上的 Townsfolk 能力信息，不受 Vortox 影响。
5. 一般阵营变化 producer 不在当前 Slice 范围内。

## deathInteraction

1. Dreamer 的目标限制是不能选择自己或 Traveller；角色页没有要求目标必须存活。
2. 目标死亡本身不改变其角色身份。
3. 本 Slice 不扩展一般死亡、复活或 dead registration 行为。
4. 如果 Philosopher 在 base Dreamer 行动前死亡，其造成的醉酒结束，因此不满足当前 exact DRUNK success path。
5. 如果 Vortox 已死亡，其能力不再强制 Dreamer 信息为假，因此不满足当前 effective-Vortox success path。

## storytellerDiscretion

1. 普通有效 Dreamer 信息要求两个角色中一个是真实角色；另一角色由说书人在相反原生类型中选择。
2. 醉酒但无有效 Vortox 时，说书人可以在合法 Dreamer 形状内给真或假信息。
3. 有效 Vortox 时，说书人不能选择包含目标真实角色的 pair。
4. 在排除目标真实角色后，说书人仍可从其他合法 GOOD 与 EVIL 角色中选择。
5. 假角色不要求必须不在场。
6. 当前 Slice 不实现自由 Storyteller 候选持久化，也不固定通用候选选择策略。

## singleTerminalOutcomeInvariant

当前 Slice 的规则与批准政策要求：

```text
one base Dreamer ability resolution
→ one terminal outcome fact
```

成功路径中的唯一事实为：

```text
ABNORMAL

causedByAnotherCharacterAbility=true
```

同时：

```text
exact canonical Philosopher-produced DRUNK
→ preserved once as positive ABILITY_IMPAIRMENT evidence
→ no second cause entry
→ no second SOURCE_DRUNKENNESS terminal fact
```

该不变量只适用于批准的 fixed first-night base-Dreamer 范围，不限制未来一般 multi-cause 模型。
