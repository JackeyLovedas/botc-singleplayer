## abilityRules

1. Dreamer is a Townsfolk. Each night, Dreamer chooses a player other than themself or a Traveller and learns exactly one GOOD character and one EVIL character.
2. Under normal effective Dreamer semantics, exactly one reported character is the target’s true character.
3. GOOD means a character whose native type is Townsfolk or Outsider. EVIL means a character whose native type is Minion or Demon. This classification is about the character, not the player’s current alignment.
4. While an effective living Vortox is in play, Townsfolk ability information must be false.
5. For Dreamer, the tested proposition is “the target is one of these two characters.” Under effective Vortox that proposition must be false, so neither the GOOD role nor the EVIL role may equal the target’s true character.
6. Vortox changes truth, not the Dreamer output shape: the result remains exactly one GOOD character and exactly one EVIL character.
7. The relevant target truth is the target’s actual current character when the Dreamer ability resolves. A change before resolution changes the truth against which both reported roles are checked; a later change does not rewrite information already delivered.
8. The official sources do not require the false characters to be out of play. A legal in-play character remains eligible if it satisfies all character-type and falsehood constraints.
9. Traveller targeting remains prohibited by the official Dreamer ability, but Travellers are outside the fixed product roster and this Slice does not add Traveller support.

## interactions

### Effective base Dreamer + effective current Vortox

The sole newly supported success is a base Dreamer whose own ability is effective at resolution, together with exactly one current Vortox whose continuous ability is effective at the same resolution. The target must be another modeled non-Traveller player. Both returned characters must be exact legal catalog characters of the required native type, and both must exclude the target’s current true character.

### False-pair semantics

For target true character `T`, a legal pair satisfies all of:

- exactly one Townsfolk-or-Outsider character;
- exactly one Minion-or-Demon character;
- GOOD character is not `T`;
- EVIL character is not `T`;
- the two selected characters are distinct catalog characters.

A GOOD target requires choosing a different GOOD character plus an EVIL character. An EVIL target requires a GOOD character plus a different EVIL character. Thus both target categories have the same final invariant: neither displayed character is the target’s true character.

### Mathematician outcome

Official and Chinese Mathematician sources count players whose abilities worked abnormally due to another character’s ability. A Vortox-forced false Dreamer result is such an abnormal result: the Dreamer player qualifies once because Vortox caused the Dreamer information to work abnormally. The count is about qualifying players, not the number of payload fields, events, role candidates or repeated facts.

For product compatibility, the existing outcome ledger must classify this terminal Dreamer result as abnormal, identify Vortox false information as the cause, and retain that the cause is another character ability. It must use the already accepted closed ledger contract. No new evidence kind or negative “no impairment” record is authorized. Existing Mathematician production semantics are unchanged; only its accepted consumption of the resulting Dreamer outcome is a regression obligation.

### Historical information

Delivered Dreamer information is a historical fact. Later target-character, target-alignment, Dreamer, Vortox or impairment changes must not recompute or rewrite the stored pair.

## drunkennessRules

Official States and Vortox sources agree:

- a drunk player has no ability;
- a drunk information character may be shown correct or incorrect apparent information under ordinary non-Vortox handling;
- if an effective Vortox is in play, Townsfolk information must still be false even when the Townsfolk player is drunk.

2B19A3A intentionally supports only an effective Dreamer source. The externally determined `source DRUNK + effective Vortox` rule is not implemented here. Governance establishes that this state is currently reachable through an accepted Philosopher path, so it is `R1_REACHABLE_BUT_EXPLICITLY_UNSUPPORTED`, not hypothetical. It must retain the already accepted retryable, receipt-free, zero-event, opportunity-stays-open behavior and is deferred to 2B19A3B.

A drunk Vortox has no functioning Vortox ability. Current-Vortox DRUNK has no accepted producer in the present repository and is `R4_FUTURE_HYPOTHETICAL_STATE`; it is not an accepted-stream implementation requirement for this Slice.

## poisoningRules

Official States and Vortox sources agree:

- a poisoned player has no ability;
- a poisoned information character may be shown correct or incorrect apparent information under ordinary non-Vortox handling;
- effective Vortox still forces Townsfolk information false even when the receiving Townsfolk is poisoned.

2B19A3A intentionally does not implement `source POISONED + effective Vortox`. Governance establishes that no accepted producer currently creates an applicable poisoned base Dreamer marker, so this is `R4_FUTURE_HYPOTHETICAL_STATE` and may be covered only as a pure rule-policy seam. A poisoned Vortox has no functioning Vortox ability. Current-Vortox POISONED likewise has no accepted producer and remains R4. No Dashii adjacency/poison derivation is out of scope and must not be inferred from absence of a represented marker.

## VortoxRules

1. Vortox’s continuous effect applies only while Vortox is alive and its ability is effective.
2. Every piece of information produced by a Townsfolk ability must be false; this includes Dreamer information.
3. Dreamer forced-false does not mean “swap which of the two is correct.” It means the entire normal Dreamer proposition is false, so both displayed characters exclude the target’s actual character.
4. The pair still contains one GOOD character and one EVIL character.
5. Effective Vortox also constrains apparent information delivered to drunk or poisoned Townsfolk, but those source-impaired paths are explicitly deferred to 2B19A3B.
6. If Vortox is dead, drunk or poisoned, its Vortox ability does not function under the general rules. Those lifecycle and impairment producers are not implemented by this Slice.
7. Missing, malformed, duplicate or conflicting proof of the current Vortox’s identity, continuity or effectiveness is not permission to assume forced-false success.

## characterChangeRules

- Character is independent of alignment.
- Dreamer learns about the target’s actual character at ability resolution. A target character change before resolution changes which character both returned roles must exclude.
- A target character change after information delivery does not alter the historical pair.
- The Dreamer source and Vortox controller must each be the required current character with effective ability at the same resolution boundary.
- General character-change production, Pit-Hag flows, gained Dreamer execution and post-delivery change producers are outside scope.

## alignmentChangeRules

An alignment change does not change a player’s character. Therefore an alignment-only change does not change:

- the target’s actual character;
- the target character excluded from both fields;
- whether a reported character belongs in the GOOD or EVIL character category;
- Dreamer’s Townsfolk character identity or Vortox’s Demon character identity.

General alignment-change production is outside scope. This evidence does not convert native character category into current player alignment.

## storytellerDiscretion

The existing simulator may continue to use its accepted stable deterministic subset of those legal choices. That subset is a product policy, not an official unique answer. Free Storyteller choice, candidate-set persistence and new choice-recording behavior are outside 2B19A3A.

## governanceTrustAndReachabilityAppendix

### Reachability classes

- `R1_CURRENTLY_REACHABLE_ACCEPTED_STREAM`: effective base Dreamer + effective current Vortox is already reachable to the current unsupported boundary and is the sole success enabled by this Slice.
- `R1_REACHABLE_BUT_EXPLICITLY_UNSUPPORTED`: source DRUNK + effective Vortox is reachable through accepted Philosopher behavior but remains unchanged, retryable, receipt-free and zero-event; it is deferred to 2B19A3B.
- `R2_LEGACY_OR_IMPORTED_ACCEPTED_HISTORY`: Dreamer V1, 2B19A1 non-actionable V2 and 2B19A2 normal V2 histories retain exact prior meaning.
- `R3_HOSTILE_OR_CORRUPTED_HISTORY`: malformed or forged persisted values, invalid cross-links, invalid atomic order and candidate shortage caused by a substituted or malformed catalog. Candidate shortage is not an R1 application scenario for a legal accepted 12-player setup.
- `R4_FUTURE_HYPOTHETICAL_STATE`: source POISONED, current-Vortox DRUNK/POISONED, ineffective-Vortox plus impaired source, No Dashii derivation, gained Dreamer execution, other-night and general lifecycle paths have no applicable accepted producer and are not accepted-stream primary tests.

### Trust classes

- `T1_EXTERNAL_OR_PERSISTED_BOUNDARY`: submitted Dreamer action, persisted Dreamer delivery, event stream/replay input and private-viewer identity. These must fail closed on hostile or noncanonical values.
- `T2_CANONICAL_DERIVED_STATE`: state derived from an accepted complete history, including current characters, current Demon, continuous role identity, represented impairments, catalog, action opportunity, target and existing outcome ledger. Only this complete canonical layer may establish current Vortox effectiveness and absence of an applicable impairment.
- `T3_MODULE_PRIVATE_PURE_CORE`: deterministic false-pair selection and private exact-value construction/comparison. This layer may consume only canonical positive inputs and must not accept caller-supplied absence claims.

### Accepted-stream pre-state rule

An incomplete prefix inside the three-event Dreamer settlement batch is not accepted history and must not be treated as one. The valid authority is the maximal accepted complete-batch prefix plus the canonical pre-delivery state produced while validating/replaying the full proposed batch. Choosing the target changes none of the current-character, Vortox, tenure, impairment or catalog inputs, so command validation, replay validation and outcome derivation can use the same canonical applicability facts without trusting a delivery’s self-assertion.

Negative impairment absence is derived from the complete canonical pre-delivery aggregate. It is not stored in the public delivery and does not require a new ledger evidence kind. State-only private projection lacks this accepted-history proof and must reject the new forced-false history; only accepted-event-stream projection may expose the already validated historical pair.
