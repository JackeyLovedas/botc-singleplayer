# Sects & Violets Role Coverage Matrix

This baseline records accepted repository implementation facts only. Its statuses are implementation coverage, not the Phase One rule-research verification labels. It is not external rule evidence and does not authorize a new slice.

Cell meanings:

- `PARTIAL: ...` — an accepted implementation covers the named path but not the full dimension.
- `SKELETON: ...` — structural metadata or a marker exists without complete behavior.
- `NOT_IMPLEMENTED` — accepted status documentation explicitly leaves the behavior out.
- `UNEVALUATED` — no accepted implementation fact was established for this dimension; this makes no claim about the actual BOTC rule.

Role-catalog inclusion alone is not behavioral implementation. No role is `COMPLETE`. Dreamer remains `PARTIAL` because Vortox forced-false behavior, Storyteller false-role selection, Traveller target restriction, and complete drunk/poison information evaluation are not implemented.

Repository evidence used for this baseline: accepted implementation status files `docs/implementation/phase-3-slice-2b1-status.md`, `phase-3-slice-2b4-status.md`, and `phase-3-slice-2b6-status.md` through `phase-3-slice-2b13-status.md`, cross-checked against current production modules. External sources were deliberately not researched in this governance update.
