# Sects & Violets Role Coverage Matrix

This baseline records accepted repository implementation facts only. Its statuses are implementation coverage, not the Phase One rule-research verification labels. It is not external rule evidence and does not authorize a new slice.

Cell meanings:

- `PARTIAL: ...` — an accepted implementation covers the named path but not the full dimension.
- `SKELETON: ...` — structural metadata or a marker exists without complete behavior.
- `NOT_IMPLEMENTED` — accepted status documentation explicitly leaves the behavior out.
- `UNEVALUATED` — no accepted implementation fact was established for this dimension; this makes no claim about the actual BOTC rule.
