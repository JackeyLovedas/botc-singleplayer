# 06 Custom Script 自定义剧本

访问日期：2026-07-06

验证状态：source-backed draft

| 问题 | 结论 |
| --- | --- |
| 自定义剧本是什么？ | 用户选择的一批允许出现角色，形成本局候选角色池。 |
| 是否等于实际配板？ | 不等于。实际配板是从角色池中选出的 12 个入场角色。 |
| 角色是否全部入场？ | 不一定。角色池通常大于玩家数。 |
| 入场角色是否必须来自角色池？ | 普通配板必须来自角色池；角色能力产生的变化需按规则单独记录。 |
| 哪些规则允许池外角色？ | 第一版应默认不允许配板阶段池外角色。麻脸巫婆等游戏中角色变化是否可指向池外角色需按角色规则和自定义剧本政策待验证。 |
| 如何检查支持12人？ | 角色池需能满足 7/2/2/1 基础构成以及方古、亡骨魔等配置修正后的类型数量。 |
| 角色池不足如何报告？ | 返回缺失类型、当前数量、需要数量、被锁定/排除冲突和建议修复。 |
| 如何导入导出？ | 导入时验证 id、名称、类型、来源、是否官方；导出需保存角色池而非实际配板。 |
| 临时排除与永久删除区别？ | 临时排除只影响本局候选，永久删除修改剧本角色池。 |
| 锁定入场与指定分配区别？ | 锁定入场保证角色在实际配板中；指定分配把某角色给具体座位。 |

## 验证规则

- `rolePool.size >= playerCount` 只是最低条件，不等于可满足类型构成。
- 类型构成验证需要在应用配置修正后重算。
- 锁定角色、排除角色、指定真人角色和指定分配要按优先级合并，并生成冲突报告。
- 不允许为了让生成成功而静默加入池外角色。

## v2 标准/沙盒边界

- 标准规则模式下，麻脸巫婆只能选择当前角色表或当前自定义剧本中的角色图标。
- 非标准沙盒模式是产品政策，不得混入标准规则引擎。
