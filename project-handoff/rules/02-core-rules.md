# 02 Core Rules 基础规则

访问日期：2026-07-06

验证状态：source-backed draft

## 建模边界

- 本项目固定为 12 名普通场上玩家，不包含旅行者、传奇或实验角色。
- 自动说书人是系统角色，不计入玩家数量、存活数、角色类型构成、提名或投票。
- 角色、阵营、登记、玩家声明和玩家认知必须分层保存。

## 胜利与失败问题

| 问题 | 建模结论 |
| --- | --- |
| 好人通常如何获胜？ | 恶魔死亡时，通常由善良阵营获胜。需先检查镜像双子等特殊胜利阻断。 |
| 邪恶通常如何获胜？ | 除特殊胜利外，若只剩两名存活玩家，邪恶阵营通常获胜。旅行者不计入本项目第一版。 |
| 恶魔死亡是否总会立即使好人获胜？ | 不是。镜像双子等能力可能阻止普通善良胜利，需进入胜利条件解析器。 |
| 只剩两名存活玩家是否总会立即使邪恶获胜？ | 通常是，但仍需检查涡流、呆瓜、镜像双子等特殊规则和同时触发。 |
| 镜像双子如何修改恶魔死亡后的胜利判断？ | 有效镜像双子存在时，善良阵营不能仅因恶魔死亡获得普通胜利；善良双子被处决时可导致邪恶即时胜利。 |
| 呆瓜如何造成特殊失败？ | 呆瓜死亡后选择一名存活玩家，若选择错误可使本队失败；系统必须记录其选择时机和可选集合。 |
| 涡流如何产生特殊胜利条件？ | 有效涡流要求镇民信息为假，并要求每天必须有处决，否则邪恶可胜利。 |
| 多个胜利条件同时触发时如何排序？ | 应作为同一事件后的胜利检查队列处理，先列出所有候选条件，再按角色特殊规则、硬规则和说书人裁量边界解析。 |
| 哪些事件之后必须立即进行胜利检查？ | 恶魔死亡、处决结算、夜间死亡结算、角色或阵营变化导致恶魔数量变化、只剩两名存活玩家、特殊失败/特殊胜利触发。 |
| 角色变化是否可能改变胜利判断？ | 可能。方古转移、麻脸巫婆创建/移除恶魔、舞蛇人交换等都可能改变恶魔身份或阵营。 |
| 阵营变化后玩家为哪个阵营获胜？ | 按当前实际阵营获胜；历史阵营和登记状态不改变最终归属。 |
| 说书人能否延迟已经满足的胜利条件？ | 不能把硬性胜利条件当作剧情裁量随意延迟；若存在同时触发或特殊角色阻断，应明确记录规则依据。 |
