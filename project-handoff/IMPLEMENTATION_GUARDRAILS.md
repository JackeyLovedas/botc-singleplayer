# Implementation Guardrails

1. 不得让 AI 直接访问 canonical state。
2. 不得使用单一全局提示词承载全部 AI 玩家。
3. 不得把角色能力全部写进巨型 if/else。
4. 不得使用单一永久夜间顺序数组。
5. 不得把醉酒或中毒写成信息取反。
6. 不得把自定义剧本写成角色分配。
7. 不得用修改真实角色模拟登记。
8. 不得把处决与死亡合并。
9. 不得在角色变化时重新执行 setup modifier。
10. 不得让自动说书人自由选择胜利方。
11. 不得把 AI 声明写入真实状态。
12. 不得让玩家通过 UI 或日志看到隐藏真值标签。
13. 不得直接让 LLM 输出成为已确认领域事件。
14. 所有 AI 行为必须先转化为候选命令并经过规则验证。
15. 所有随机行为必须记录随机种子和选择候选。
16. 所有说书人裁量必须记录合法候选和最终选择。
17. 所有 `PARTIAL` 角色在实现前必须补充测试。
18. 规则冲突先查 `RULES_BASELINE.md`，再查对应规则文件和测试。
19. 遇到规则未定义时停止该功能并记录问题，不用实现细节掩盖规则缺口。
